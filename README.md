what does git clone do; what does github fork do. Difference  between git clone and github fork.

Git clone downloads a copy of a repository from GitHub to your local computer. After cloning, you can open the files, edit them, commit changes, and push them back if you have permission.

GitHub fork creates a copy of someone else’s repository under your own GitHub account. It lets you make changes safely without affecting the original repository, and you can later send a pull request if you want the original owner to review and merge your changes.

The main difference is that clone is local and fork is online. A fork copies a repository on GitHub to your GitHub account, while clone copies a repository from GitHub to your computer. For example, in group work, you may fork a classmate’s repository first, then clone your fork to your computer to edit it.
